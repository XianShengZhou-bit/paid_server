# backend 文件架构

```text
backend/
├── 后端业务流程文档.md          # 六个 API、处理流程、登录态、qr_token 说明
├── common/                      # 公共基础设施（已实现）
│   ├── config.hpp               # 读取 .env，提供 MySQL / Redis / JWT / 端口等配置
│   ├── jwt.hpp                  # qr_token 签发与验签；邮箱、身份证 HMAC 哈希
│   ├── logger.hpp               # spdlog 日志封装，敏感字段脱敏
│   ├── mysql_pool.hpp           # MySQL 连接池；users / orders / accounts / payment_sessions / payment_attempts 的 DAO
│   └── password_verifier.hpp    # 登录密码 HMAC-SHA256 校验（对比 users.password_hash）
├── payment_impl.hpp             # 六个 API 的声明与实现，直接调用 common/
└── main.cpp                     # 进程入口：读配置、初始化日志/连接池、启动 HTTP 服务
```

## 各文件职责

| 文件 | 职责 |
| :--- | :--- |
| `后端业务流程文档.md` | 网关转发的 6 个接口定义、逐步处理流程、Cookie 登录态约定、qr_token 载荷说明 |
| `common/config.hpp` | 加载 `.env`；暴露 `MySqlConfig`、`JwtConfig`、`PaymentBackendConfig` 等配置结构 |
| `common/jwt.hpp` | 生成/校验支付二维码 `qr_token`（JWT）；邮箱、身份证的 HMAC 哈希与脱敏辅助 |
| `common/logger.hpp` | 初始化 spdlog；按级别写文件/控制台；日志中屏蔽密码、token 等敏感信息 |
| `common/mysql_pool.hpp` | 连接池管理；查用户/订单/账号/支付会话/支付尝试；创建会话、实名更新、支付成功事务 |
| `common/password_verifier.hpp` | 用 `PASSWORD_HASH_SECRET` 对明文登录密码做 HMAC-SHA256，与库中 `password_hash` 比对 |
| `payment_impl.hpp` | 请求/响应结构体、六个 API 处理函数；按业务流程文档编排，直接调用 `common/` |
| `main.cpp` | `main()`：加载 `.env`、初始化 logger 与 MySQL 连接池、注册路由、监听指定端口 |

## 六个 API 直接依赖的 common

| API | 直接调用的 common |
| :--- | :--- |
| `GET /api/payment/orders/{order_sn}` | `mysql_pool.hpp` |
| `POST /api/payment/confirm` | `mysql_pool.hpp`、`jwt.hpp` |
| `POST /api/users/real-auth/complete` | `mysql_pool.hpp`、`jwt.hpp` |
| `GET /api/payment/page?token=` | `jwt.hpp`、`mysql_pool.hpp` |
| `POST /api/payment/attempt/init` | `jwt.hpp`、`mysql_pool.hpp` |
| `POST /api/payment/pay` | `jwt.hpp`、`mysql_pool.hpp`、`password_verifier.hpp` |

`main.cpp` 只负责启动与初始化各组件；业务逻辑全在 `payment_impl.hpp`。

业务流程以 `后端业务流程文档.md` 为准；表结构见仓库根目录 `数据库表结构/`。
