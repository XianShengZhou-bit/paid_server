# 页面 A：confirm 与 WebSocket 时序

1. 进入页面 → `GET /api/payment/orders/{order_sn}`（只拉订单，还没有二维码）
2. 用户勾选「我已确认」→ 点「继续支付」→ `POST /api/payment/confirm`
3. 同一次 HTTP 响应带回：`qr_payload`、`websocket_url`、`payment_session_id` 等
4. 前端先存 `qr_payload`，再 `new WebSocket(url)`（新 TCP + HTTP Upgrade）
5. Gateway 握手成功 → 推 `BIND_SUCCESS`
6. 收到 `BIND_SUCCESS` 后，才用第 3 步存好的 `qr_payload` 渲染二维码
