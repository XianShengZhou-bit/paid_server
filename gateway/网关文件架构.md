# gateway 文件架构

```text
gateway/
├── 网关业务流程文档.md          # HTTP 转发、登录态、WebSocket 绑定与推送说明
├── common/                      # 公共基础设施（待实现）
│   ├── config.hpp               # 读取 .env，提供网关/后端地址、Redis、WebSocket 参数
│   ├── logger.hpp               # spdlog 日志封装，敏感字段脱敏
│   ├── session_store.hpp        # session_id → user_id 会话存储（当前 map，后续 Redis）
│   └── http_forwarder.hpp       # 将请求转发到内网支付后端
├── gateway_impl.hpp             # 鉴权、路由转发、WebSocket 会话与支付结果推送
└── main.cpp                     # 进程入口：HTTP + WebSocket 双监听
```

## 各文件职责

| 文件 | 职责 |
| :--- | :--- |
| `网关业务流程文档.md` | 对外 HTTP/WebSocket 入口、登录态、转发规则、推送协议 |
| `common/config.hpp` | 加载 `.env`；暴露 `PaymentGatewayConfig`、`PaymentBackendConfig`、`RedisConfig` 等 |
| `common/logger.hpp` | 初始化 spdlog；按级别写文件/控制台；日志中屏蔽密码、token 等敏感信息 |
| `common/session_store.hpp` | 维护 `session_id → { user_id }`；登录写入、请求校验、登出删除 |
| `common/http_forwarder.hpp` | 将 `/api/*` 请求原样转发到 `PAYMENT_BACKEND_IP:PAYMENT_BACKEND_HTTP_PORT` |
| `gateway_impl.hpp` | 登录/登出、Cookie 鉴权、按路径转发、WebSocket 绑定、`PAYMENT_RESULT` 推送 |
| `main.cpp` | `main()`：加载 `.env`、初始化 logger 与会话存储、启动 HTTP（9993）与 WebSocket（9992） |

## 网关直接处理的入口

| 入口 | 方法 | 处理方式 |
| :--- | :--- | :--- |
| `/api/auth/login` | POST | 网关校验账号密码，写 session，Set-Cookie |
| `/api/auth/logout` | POST | 网关删 session，清 Cookie |
| `/api/payment/orders/{order_sn}` | GET | 验 Cookie → 加 `X-User-Id` → 转发后端 |
| `/api/payment/confirm` | POST | 验 Cookie → 加 `X-User-Id` → 转发后端 |
| `/api/users/real-auth/complete` | POST | 验 Cookie → 加 `X-User-Id` → 转发后端 |
| `/api/payment/page` | GET | 直接转发后端（凭 `qr_token`，不走 Cookie） |
| `/api/payment/attempt/init` | POST | 直接转发后端 |
| `/api/payment/pay` | POST | 直接转发后端 |
| `/ws/payment` | WebSocket | 网关维护连接；绑定 `payment_session_id`；推送 `PAYMENT_RESULT` |
| `/internal/payment/result` | POST | 内网接收后端支付成功事件，触发 WebSocket 推送 |

`main.cpp` 只负责启动与初始化；业务逻辑全在 `gateway_impl.hpp`。

业务流程以 `网关业务流程文档.md` 为准；支付业务细节见 `backend/后端业务流程文档.md`。
