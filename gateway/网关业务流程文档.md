浏览器经网关访问 API（示例 `http://127.0.0.1:9993`），页面资源由 Nginx 提供（示例 `http://127.0.0.1:9994`）。网关将 `/api/*` 转发到内网支付后端（`PAYMENT_BACKEND_IP:PAYMENT_BACKEND_HTTP_PORT`）。响应统一为 `{"code":0,"message":"","data":{}}`。

| 方法      | 路径                                  | 请求体                                                | 说明                              |
| :-------- | :------------------------------------ | :---------------------------------------------------- | :-------------------------------- |
| POST      | `/api/auth/login`                     | `{"username":"","password":""}`                       | 登录，写 `session_id` Cookie      |
| POST      | `/api/auth/logout`                    | —                                                     | 登出，清 Cookie                   |
| GET       | `/api/payment/orders/{order_sn}`      | —                                                     | 转发后端（需登录态）              |
| POST      | `/api/payment/confirm`                | `{"order_sn":"","agreement_checked":true}`            | 转发后端（需登录态）              |
| POST      | `/api/users/real-auth/complete`       | `{"email":"","id_card":"","return_url":""}`           | 转发后端（需登录态）              |
| GET       | `/api/payment/page?token={qr_token}`  | —                                                     | 转发后端（扫码支付，不需 Cookie） |
| POST      | `/api/payment/attempt/init`           | `{"qr_token":""}`                                     | 转发后端                          |
| POST      | `/api/payment/pay`                    | `{"request_id":"","qr_token":"","login_password":""}` | 转发后端                          |
| WebSocket | `/ws/payment?payment_session_id={id}` | —                                                     | 页面 A 绑定会话，等待支付结果     |

## 当收到浏览器请求后，网关对各入口的处理流程：

**`POST /api/auth/login`**

1. 校验 `username` / `password` 非空。
2. 查 `users` 校验账号密码（`password_hash`）。
3. 生成 `session_id`，写入会话存储（当前 map，后续 Redis）。
4. `Set-Cookie: session_id=...; HttpOnly; SameSite=Lax`。
5. 返回登录成功。

**`POST /api/auth/logout`**

1. 读 Cookie 中 `session_id`。
2. 删除会话存储中的记录。
3. 清除 Cookie。
4. 返回成功。

**需登录态的 `/api/*`（orders / confirm / real-auth）**

1. 读 Cookie 中 `session_id`。
2. 查会话存储，无效则返回 `40101`。
3. 转发请求到内网后端，附加请求头 `X-User-Id: {user_id}`。
4. 将后端响应原样返回浏览器。

**不需登录态的 `/api/*`（page / attempt/init / pay）**

1. 不校验 Cookie。
2. 原样转发到内网后端（保留 method、path、query、body）。
3. 将后端响应原样返回浏览器。

**`GET /ws/payment?payment_session_id={id}`**

1. 完成 WebSocket 握手。
2. 校验 `payment_session_id` 格式
3. 建立映射：`payment_session_id -> connection`。
4. 向客户端推送：

```json
{
  "type": "BIND_SUCCESS"
}
```

5. 页面 A 收到 `BIND_SUCCESS` 后再渲染二维码。

**`POST /internal/payment/result`（内网，后端调用）**

1. 解析 body：`type`、`payment_session_id`、`order_sn`、`payment_status` 等。
2. 按 `payment_session_id` 查找 WebSocket 连接。
3. 推送：

```json
{
  "type": "PAYMENT_RESULT",
  "payment_session_id": "PS...",
  "order_sn": "ORD...",
  "payment_status": "PAID",
  "message": "支付成功"
}
```

4. 找不到连接则记录日志（可返回 `50003` 给后端）。

## 整个业务流程（网关视角）：

1. **页面 A**：浏览器带 Cookie 调 `GET /api/payment/orders/{order_sn}` → 网关验 session → 转发后端。
2. **继续支付**：`POST /api/payment/confirm` → 网关验 session → 转发后端 → 返回 `qr_payload`、`websocket_url`。
3. **WebSocket**：页面 A 连 `ws://网关:9992/ws/payment?payment_session_id=...` → 网关回 `BIND_SUCCESS`。
4. **页面 B**：手机经 Nginx 打开扫码页 → 浏览器调 `/api/payment/page`、`attempt/init`、`pay` → 网关直接转发后端。
5. **支付成功**：后端事务完成后 `POST /internal/payment/result` → 网关向页面 A 推送 `PAYMENT_RESULT`。

## 电脑端如何保持登录态：

适用页面 A、C（同域 Cookie）；页面 B 扫码不走此机制，凭 `qr_token`。

1. **登录**：`POST /api/auth/login` → 网关写 session → Set-Cookie。
2. **后续请求**：浏览器自动带 Cookie → 网关验 session → 转发时加 `X-User-Id`。
3. **后端**：不读 Cookie，只信 `X-User-Id`。
4. **登出**：`POST /api/auth/logout` → 删 session → 清 Cookie。

## 部署与端口：

| 组件           | bind 地址           | 对外示例              | 职责             |
| :------------- | :------------------ | :-------------------- | :--------------- |
| 网关 HTTP      | `0.0.0.0`（硬编码） | `127.0.0.1:9993`      | API 入口、登录态 |
| 网关 WebSocket | `0.0.0.0`（硬编码） | `ws://127.0.0.1:9992` | 支付结果推送     |
| Nginx          | `0.0.0.0`（硬编码） | `127.0.0.1:9994`      | 前端页面         |
| 支付后端       | 内网                | `localhost:9990`      | 业务逻辑，不对外 |

`.env` 中 `PAYMENT_GATEWAY_IP`、`PAYMENT_NGINX_IP` 为返回给客户端的可访问地址，不是 bind 地址。

## WebSocket 推送消息类型：

| `type`           | 方向          | 含义                       |
| :--------------- | :------------ | :------------------------- |
| `BIND_SUCCESS`   | 网关 → 页面 A | 会话绑定成功，可展示二维码 |
| `PAYMENT_RESULT` | 网关 → 页面 A | 支付完成通知               |

支付事实以 MySQL 事务为准；WebSocket 只负责通知，不存储支付结果。
