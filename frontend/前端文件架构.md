# frontend 文件架构

```text
frontend/
├── 前端业务流程文档.md          # 三页面 URL、经网关的 API/WebSocket、页面内处理流程
├── 前端文件架构.md              # 本文件
├── nginx.conf.template          # Nginx 配置模板（由 run_all.sh 渲染后启动）
├── pages/                       # 静态页面（由 Nginx 托管）
│   ├── payment_confirm.html     # 页面 A：订单确认、继续支付、二维码、等支付结果
│   ├── payment_scan.html        # 页面 B：扫码支付
│   ├── login.html               # 登录页
│   └── real_auth_complete.html  # 页面 C：实名补全
└── common/                      # 公共脚本
    ├── config.js                # 网关 API 基址、WebSocket 基址
    └── api.js                   # 封装 fetch（含 Cookie）、WebSocket 连接与消息解析
```

## 各文件职责

| 文件 | 职责 |
| :--- | :--- |
| `前端业务流程文档.md` | 三页面入口 URL、经网关的请求时序、WebSocket 消息、业务码处理 |
| `pages/payment_confirm.html` | 页面 A：读 `order_sn`、拉订单、`confirm`、连 WS、展示二维码与支付结果 |
| `pages/payment_scan.html` | 页面 B：读 `qr_token`、`page` → `attempt/init` → `pay` |
| `pages/real_auth_complete.html` | 页面 C：读 `return_url`、提交实名、成功后跳回 |
| `common/config.js` | `GATEWAY_HTTP_ORIGIN`、`GATEWAY_WS_ORIGIN` 等可配置常量 |
| `common/api.js` | 统一 `fetch` 封装（`credentials: "include"`）、解析 `{code,message,data}` |

## 各页面经网关的入口

| 页面 | Nginx 路径 | 经网关调用 |
| :--- | :--- | :--- |
| A | `/payment/confirm` | `GET /api/payment/orders/{order_sn}`、`POST /api/payment/confirm`、`WebSocket /ws/payment` |
| C | `/user/real-auth/complete` | `POST /api/users/real-auth/complete` |
| B | `/payment/scan` | `GET /api/payment/page`、`POST /api/payment/attempt/init`、`POST /api/payment/pay` |

页面只负责展示与发起请求；业务校验、支付事务、推送由网关与后端完成。

业务流程以 `前端业务流程文档.md` 为准；API 细节见 `gateway/网关业务流程文档.md`、`backend/后端业务流程文档.md`。
