Nginx 提供静态页面（示例 `http://127.0.0.1:9994`）；页面内 JS 经网关访问 API（示例 `http://127.0.0.1:9993`）、WebSocket（示例 `ws://127.0.0.1:9992`）。响应统一解析 `{"code":0,"message":"","data":{}}`。

| 页面 | Nginx 路径 | 经网关的请求 | 登录态 |
| :--- | :--- | :--- | :--- |
| A 订单确认 | `/payment/confirm?order_sn={order_sn}` | `GET /api/payment/orders/{order_sn}`、`POST /api/payment/confirm`、`WebSocket /ws/payment?payment_session_id={id}` | Cookie |
| C 实名补全 | `/user/real-auth/complete?return_url={return_url}` | `POST /api/users/real-auth/complete` | Cookie |
| B 扫码支付 | `/payment/scan?token={qr_token}` | `GET /api/payment/page`、`POST /api/payment/attempt/init`、`POST /api/payment/pay` | 无（凭 `qr_token`） |

## 各页面处理流程

**页面 A（`/payment/confirm`）**

1. 从 URL 读取 `order_sn`。
2. `GET /api/payment/orders/{order_sn}` 拉订单与账号信息并渲染。
3. 用户勾选声明后点「继续支付」→ `POST /api/payment/confirm`，body：`{"order_sn":"","agreement_checked":true}`。
4. `code === 40910`：跳转页面 C，`return_url` 设为 `/payment/confirm?order_sn={order_sn}`。
5. `code === 0`：用 `data.websocket_url` 建立 WebSocket；收到 `BIND_SUCCESS` 后，用 `data.qr_payload` 渲染二维码。
6. 收到 `PAYMENT_RESULT` 且 `payment_status === "PAID"`：展示支付成功。

**页面 C（`/user/real-auth/complete`）**

1. 从 URL 读取 `return_url`（由页面 A 跳转时拼接，如 `/payment/confirm?order_sn=ORD001`）。
2. 用户提交邮箱、身份证号 → `POST /api/users/real-auth/complete`，body：`{"email":"","id_card":"","return_url":""}`。
3. `code === 0`：浏览器跳转 `return_url`（回到页面 A）。

**页面 B（`/payment/scan`）**

1. 从 URL 读取 `qr_token`（手机扫码打开，不走 Cookie）。
2. `GET /api/payment/page?token={qr_token}` 拉支付页数据。
3. `POST /api/payment/attempt/init`，body：`{"qr_token":""}`，取得 `request_id` 并暂存（如 `sessionStorage`）。
4. 用户输入登录密码并确认 → `POST /api/payment/pay`，body：`{"request_id":"","qr_token":"","login_password":""}`。
5. `code === 0`：展示支付成功。

## 整个业务流程（前端视角）

1. **页面 A**：Nginx 打开确认页 → 拉订单 → `confirm` → 连 WebSocket → 展示二维码。
2. **页面 C**（若未实名）：补全实名 → 回 **A** 重新 `confirm`。
3. **页面 B**：扫码打开 → `page` → `attempt/init` → `pay`。
4. **支付成功**：B 展示结果；A 经 WebSocket 收 `PAYMENT_RESULT`。

## 登录态

页面 A、C 依赖网关 `session_id` Cookie（`credentials: "include"`）；页面 B 凭 URL 中的 `qr_token`，不带 Cookie。

## WebSocket 消息（页面 A）

| `type` | 处理 |
| :--- | :--- |
| `BIND_SUCCESS` | 渲染二维码 |
| `PAYMENT_RESULT` | 读 `order_sn`、`payment_status`、`message`，展示支付结果 |

## 前端需处理的业务码

| `code` | 处理 |
| :--- | :--- |
| `0` | 成功，读 `data` |
| `40101` | 登录态无效，引导重新登录 |
| `40910` | 跳转页面 C |

业务流程以本文档为准；网关转发规则见 `gateway/网关业务流程文档.md`。
