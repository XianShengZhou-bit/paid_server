## 一、电脑登录

1. 打开 `/login`，提交用户名密码 → `POST /api/auth/login`
2. 网关转发后端；后端校验用户与密码
3. 生成新的 `session_id`，Redis：`payment:session:{session_id} → user_id`
4. 经网关透传 `Set-Cookie: session_id=...`
5. 之后需登录接口：浏览器带 Cookie → 网关 Redis 换成 `user_id` → 转发时加 `X-User-Id`
6. 允许多端登录：再登录会再发一个新 `session_id`，**不覆盖、不踢掉**旧会话

---

## 二、页面 A：看订单

1. 打开 `/payment/confirm?order_sn=...`（带 Cookie）
2. `GET /api/payment/orders/{order_sn}`
3. 后端用 `X-User-Id` 查订单，**校验 `buyer_id` 一致**，否则 `40401`
4. 前端渲染订单/账号信息

---

## 三、页面 A：继续支付 → 出码

1. 勾选协议 → 点「继续支付」→ `POST /api/payment/confirm`
2. 后端校验：协议、用户状态、实名（未实名 → `40910` 去页面 C）、订单归属与待支付、账号可交易
3. **创建或复用**该订单的 `payment_session_id`（WaitingPay 未过期则复用）
4. **新签发** `qr_token`（JWT，内含 `payment_session_id` 等，不落库）
5. **同一次 HTTP 响应**返回：
   - `payment_session_id`
   - `qr_payload`（即扫码 URL：`/payment/scan?token=...`）
   - `websocket_url`
6. 前端用 `websocket_url` 连网关，HTTP 升级为 WebSocket
7. 网关绑定 `payment_session_id → 连接`，推送 **`BIND_SUCCESS`**
8. 前端收到 `BIND_SUCCESS` 后，用**已有的** `qr_payload` 在本地画二维码（不再单独向服务器要一次码）

（若未实名：页面 C 补全 → 回 A → 重新点继续支付。）

## 四、手机：扫码支付（无 Cookie）

1. 扫码打开 `qr_payload` 里的 URL（谁拿到码都能打开页面）
2. 自动两步：
   - `GET /api/payment/page?token=qr_token`：验 JWT、查会话/订单/账号，返回展示数据
   - `POST /api/payment/attempt/init`：再校验后**新生成** `request_id`，写入 `payment_attempts`，返回手机
3. 用户输入**买家登录密码** → `POST /api/payment/pay`，body：  
   `request_id` + `qr_token` + `login_password`
4. 后端：
   - 按 `request_id` 做同次提交幂等
   - 验 `qr_token` → 得到 `payment_session_id` → 查出会话上的 `order_sn` / `buyer_id`
   - 校验 attempt 绑定、订单/账号/用户、密码
   - `finalizePaymentSuccess` 事务落库（订单已付、账号已售、会话已付、attempt 成功）
5. 后端调网关 `POST /internal/payment/result`
6. 网关按 `payment_session_id` 给电脑 A 推 **`PAYMENT_RESULT`**
7. 电脑 A 渲染支付成功；手机看 `pay` 的 HTTP 成功响应

## 五、关系小结（防混淆）

```text
登录 Cookie session_id  ──Redis──► user_id（仅电脑端鉴权）
confirm ──► payment_session_id（一单一会话，可复用）
         ──► qr_token / qr_payload（每次新签，画进二维码）
扫码页 init ──► request_id（每次进页新建 attempt；不在二维码里）
pay：qr_token → 会话 → 订单；request_id → 幂等；密码 → 买家身份
一笔订单只成功一次：靠事务行锁；request_id 管同一次点击的重试
```

---

## 六、一句话串起来

电脑登录拿 `session_id` → 带 Cookie 拉单并校验买家 → confirm 同时拿到会话 ID、扫码 URL、WS 地址 → WS `BIND_SUCCESS` 后画码 → 手机扫码拿 `request_id` → 密码支付落库 → 网关 WS 通知电脑成功。
